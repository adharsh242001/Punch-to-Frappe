# Product Team Agents

Use this skill to select and coordinate specialist product agents for software/product work. Keep the active team small: choose only the roles needed for the current decision or deliverable.

## Quick Start

1. Identify the user's objective: strategy, design, implementation, launch, growth, support, or operations.
2. Pick the minimum set of relevant agents from `references/agent-catalog.md`.
3. Assign each agent a concrete output and boundary.
4. Combine their outputs into one executable plan or artifact.
5. Escalate final tradeoffs to the Founder / Orchestrator Agent when priorities conflict.

## Agent Selection

Use these default groupings:

- New product idea: Vision & Product Strategy, UI/UX Design, Founder / Orchestrator.
- App or web build: Frontend Engineering, Backend Engineering, Database & Data Engineering, QA & Testing.
- Production deployment: DevOps & Cloud Infrastructure, Security, Backend Engineering, QA & Testing.
- AI feature: AI & Automation, Backend Engineering, Product Analytics, Security.
- Growth or monetization: Marketing & Growth, Product Analytics, Finance & Monetization, Vision & Product Strategy.
- User retention: Community & Support, Product Analytics, UI/UX Design, Founder / Orchestrator.

## Coordination Rules

- Start with the user problem and desired outcome before tools or features.
- Prefer MVP scope unless the user explicitly asks for a full roadmap.
- Name assumptions and risks clearly.
- Convert specialist recommendations into owner, deliverable, priority, and next action.
- Avoid assigning every agent by default; too many roles create noise.
- When engineering work is requested, turn strategy/design recommendations into concrete implementation tasks.
